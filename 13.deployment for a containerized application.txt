# 🔷 Practical 13 — Deep Explanation

## **Create deployment & verify pod status**

---

## 🔹 Big Picture (What you’re doing)

You ask Kubernetes:

> “Run my application (nginx) and keep it running.”

Kubernetes then:

1. Creates a **Deployment**
2. Deployment creates a **ReplicaSet**
3. ReplicaSet creates a **Pod**
4. Pod runs the **Container (nginx)**

👉 Chain:
**Deployment → ReplicaSet → Pod → Container**

---

## 🔹 Step 1: Create Deployment

```bash
kubectl create deployment status-app --image=nginx
```

### ✔ What happens internally?

* Deployment object is created
* Kubernetes:

  * Pulls nginx image
  * Creates 1 ReplicaSet
  * ReplicaSet creates 1 Pod
  * Pod starts container

### ✔ Expected output:

```bash
deployment.apps/status-app created
```

---

## 🔹 Step 2: Check Pods

```bash
kubectl get pods
```

### ✔ What this shows:

* Pod name
* READY → container running or not
* STATUS → current state

---

### 🔸 Possible statuses:

| Status           | Meaning              |
| ---------------- | -------------------- |
| Pending          | Pod is being created |
| Running ✅        | Everything working   |
| ImagePullBackOff | Image not found      |
| CrashLoopBackOff | App crashing         |

---

### ✔ Correct output:

```bash
status-app-xxxxx   1/1   Running
```

👉 Meaning:

* 1 container inside pod
* 1 is running

---

## 🔹 Step 3: Describe Pods (VERY IMPORTANT)

```bash
kubectl describe pods
```

### ✔ What it shows:

Detailed info like:

* Node where pod is running
* Image used
* Events (errors if any)
* Container state

---

### ✔ Key parts to understand:

#### 1. Status

```bash
Status: Running
```

✔ Pod is healthy

---

#### 2. Container Info

```bash
Containers:
  nginx:
    Image: nginx
    State: Running
```

✔ Container started successfully

---

#### 3. Events (MOST IMPORTANT for errors)

```bash
Events:
  Pulled image
  Created container
  Started container
```

✔ Shows full lifecycle

---

## 🔹 Step 4: Check Deployment

```bash
kubectl get deployments
```

### ✔ Output:

```bash
status-app   1/1   1   1
```

### ✔ Meaning:

| Column     | Meaning             |
| ---------- | ------------------- |
| READY      | running pods        |
| UP-TO-DATE | latest version pods |
| AVAILABLE  | working pods        |

👉 `1/1` = success

## 🔹 Common Errors (VERY IMPORTANT)

### ❌ Image issue

```bash
ImagePullBackOff
```

👉 Fix: check image name

---

### ❌ Pod not starting

```bash
CrashLoopBackOff
```

👉 Fix: check logs

```bash
kubectl logs <pod-name>
```

---

### ❌ Pod stuck

```bash
Pending
```

👉 Fix:

* Wait
* Check resources
