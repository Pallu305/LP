# **12. Implement handlers in an Ansible playbook and explain their role during execution**

---

# **Aim**

To implement handlers in an Ansible playbook and understand their role in efficient task execution.

---

# **Theory**

In Ansible, **handlers** are special tasks that run **only when triggered (notified)** by another task.

### 🔹 Why Handlers?

* Avoid unnecessary service restarts
* Improve efficiency
* Execute actions **only when changes occur**

---

### 🔹 Key Concepts

* **notify** → triggers handler
* **handlers section** → defines handler tasks
* **runs once at end of play**
* **executes only if change happens**

---

### 🔹 Example Flow

1. A task modifies a configuration file
2. That task uses `notify`
3. Handler runs **only if file was changed**
4. Service restarts only when required

---

# **Requirements**

* WSL (Ubuntu)
* Ansible installed

---

# **Steps**

---

## **1. Open WSL**

```bash
wsl
```

---

## **2. Create Project Folder**

```bash
mkdir ansible-handler
cd ansible-handler
```

---

## **3. Create Inventory File**

```bash
vim inventory
```

Paste:

```bash
[web]
localhost ansible_connection=local
```

Save:

```
ESC → :wq
```

---

## **4. Create Playbook**

```bash
vim handler.yml
```

Paste:

```yaml
---
- name: Demonstrate Handlers in Ansible
  hosts: web
  become: yes

  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
        update_cache: yes

    - name: Change Apache Port to 8081
      lineinfile:
        path: /etc/apache2/ports.conf
        regexp: '^Listen 80'
        line: 'Listen 8081'
      notify: Restart Apache

  handlers:
    - name: Restart Apache
      service:
        name: apache2
        state: restarted
```

Save:

```
ESC → :wq
```

---

## **5. Run Playbook**

```bash
ansible-playbook -i inventory handler.yml --ask-become-pass
```

---

# **Expected Output (First Run)**

```
PLAY [Demonstrate Handlers in Ansible] ***********************

TASK [Install Apache] ***************************************
changed: [localhost]

TASK [Change Apache Port to 8081] ***************************
changed: [localhost]

RUNNING HANDLER [Restart Apache] ****************************
changed: [localhost]

PLAY RECAP **************************************************
localhost : ok=2  changed=2  failed=0
```

---

# **Expected Output (Second Run)**

```
TASK [Change Apache Port to 8081] ***************************
ok: [localhost]

(no handler executed)
```

👉 This proves **idempotency + handler working**

---

# **6. Verify Output**

Open:

```
http://localhost:8081
```

---

# **Result**

Handlers were successfully implemented. Apache restarted **only when configuration changed**.

---

# **Conclusion**

* Handlers improve efficiency
* Prevent unnecessary service restarts
* Work based on **event-driven execution**
* Ensure optimized automation

---

# 🧠 **Important Viva Points**

* Handlers run **only when notified**
* Executed **at end of play**
* Used mainly for **service restart/reload**
* Improve performance and avoid redundancy
