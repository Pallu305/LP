#2 Preprocessing
# Select relevant numerical columns for clustering
features = ['QUANTITYORDERED', 'PRICEEACH', 'SALES', 'MSRP']
X = df[features].dropna()

# Standardize the data so larger numbers (like SALES) don't dominate the distance calculations
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)



#3 The Elbow Method
wcss = [] # Within-Cluster Sum of Squares

# Test K values from 1 to 10
for i in range(1, 11):
    kmeans = KMeans(n_clusters=i, init='k-means++', random_state=42, n_init=10)
    kmeans.fit(X_scaled)
    wcss.append(kmeans.inertia_)

# Plot the Elbow Graph
plt.figure(figsize=(8, 5))
plt.plot(range(1, 11), wcss, marker='o', linestyle='--', color='b')
plt.title('The Elbow Method')
plt.xlabel('Number of Clusters (K)')
plt.ylabel('WCSS')
plt.grid(True)
plt.show()
    





# 4 Apply K-Means with Optimal K
# Assuming the elbow drops and flattens around K=3 (adjust based on your plot's actual elbow)
optimal_k = 3 
kmeans_final = KMeans(n_clusters=optimal_k, init='k-means++', random_state=42, n_init=10)

# Add the cluster labels back to our dataframe
X['Cluster'] = kmeans_final.fit_predict(X_scaled)


print(f"--- K-Means Clustering Completed (K={optimal_k}) ---")
print("Number of data points in each cluster:")
print(X['Cluster'].value_counts())






plt.scatter(X['SALES'], X['QUANTITYORDERED'], c=X['Cluster'], cmap='viridis')
plt.xlabel('SALES')
plt.ylabel('QUANTITYORDERED')
plt.title('K-Means Clustering')
plt.show()

# 🔷 1. Problem Type

* This is **Unsupervised Learning**
* No target variable → finding patterns/groups in data

👉 Goal: **Group similar sales records into clusters**

---

# 🔷 2. Load Dataset

```python
df = pd.read_csv("sales_data_sample.csv", encoding='latin1')
```

* Dataset contains:

  * Sales transactions
  * Product + customer details

---

# 🔷 3. Feature Selection

```python
features = ['QUANTITYORDERED', 'PRICEEACH', 'SALES', 'MSRP']
X = df[features].dropna()
```

✔ Only numerical columns are selected
✔ Missing values removed

👉 These features represent:

* Quantity → demand
* Price → cost
* Sales → revenue
* MSRP → listed price

---

# 🔷 4. Feature Scaling

```python
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

### Why important?

* K-Means uses **distance**
* Large values (like SALES) can dominate

👉 Scaling makes all features equal importance

---

# 🔷 5. Elbow Method (Find Best K)

```python
for i in range(1, 11):
    kmeans.fit(X_scaled)
    wcss.append(kmeans.inertia_)
```

### Concept:

* **WCSS (Within Cluster Sum of Squares)** = total distance of points from cluster center

👉 Goal:

* Minimize WCSS
* But not too many clusters

### Elbow Graph:

* X-axis → number of clusters (K)
* Y-axis → WCSS

👉 Choose K where graph **bends (elbow point)**

✔ In your case: **K = 3**

---

# 🔷 6. K-Means Clustering

```python
kmeans_final = KMeans(n_clusters=3)
X['Cluster'] = kmeans_final.fit_predict(X_scaled)
```

### Concept:

K-Means works in steps:

1. Choose K centroids
2. Assign points to nearest centroid
3. Update centroids
4. Repeat until stable

---

# 🔷 7. Cluster Output

```python
X['Cluster'].value_counts()
```

👉 Shows:

* How many data points in each cluster

---

# 🔷 8. Visualization

```python
plt.scatter(X['SALES'], X['QUANTITYORDERED'], c=X['Cluster'])
```

👉 Each color = one cluster

### Interpretation:

* Cluster 0 → low sales
* Cluster 1 → medium sales
* Cluster 2 → high sales

---

# 🔷 9. Hierarchical Clustering

```python
linked = linkage(X_scaled, method='ward')
```

### Concept:

* Builds clusters step-by-step (tree structure)

---

# 🔷 10. Dendrogram

```python
dendrogram(linked)
```

### Interpretation:

* Each merge = combining clusters
* Height = distance

👉 Cut the tree at a level → get clusters

---

# 🔷 🔥 Key Differences

| K-Means       | Hierarchical      |
| ------------- | ----------------- |
| Fast          | Slower            |
| Needs K       | No need initially |
| Flat clusters | Tree structure    |
| Large data    | Small/medium data |

# 🔷 11. Important Concepts

### ✔ WCSS

### ✔ Centroid

* Center of cluster

### ✔ Distance

* Usually Euclidean distance

### ✔ Ward Method

* Minimizes variance within clusters
