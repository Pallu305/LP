Here is the **complete practical write-up** for:

# **8. Create an inventory file and demonstrate variable usage and task execution in Ansible**

---

# **Aim**

To create an Ansible inventory file and demonstrate the use of variables in a playbook for task execution.

---

# **Theory**

In Ansible, an **inventory file** defines the list of target hosts where tasks will be executed.

### Key Concepts

* **Inventory**: Contains host details and grouping
* **Variables (vars)**: Used to store values and reuse them dynamically
* **Playbook**: Executes tasks using defined variables
* **Templating**: Variables are accessed using `{{ variable_name }}`

### Why Variables?

* Avoid hardcoding values
* Improve flexibility and reusability
* Make playbooks dynamic and scalable

### Working Flow

1. Inventory defines the target host
2. Variables are declared inside playbook
3. Tasks use variables dynamically
4. Execution happens on target system

---

# **Requirements**

* WSL (Ubuntu)
* Ansible installed

---

# **Steps**

---

## **1. Open WSL (Ubuntu)**

```
wsl
```

---

## **2. Create Project Directory**

```
mkdir ansible-variable
cd ansible-variable
```

---

## **3. Create Inventory File**

```
vim inventory
```

Paste:

```
[web]
localhost ansible_connection=local ansible_user=ubuntu
```

Save:

```
ESC → :wq
```

---

## **4. Create Playbook**

```
vim variable.yml
```

Paste:

```
---
- name: Demonstrate Variables in Ansible
  hosts: web
  become: yes

  vars:
    package_name: nginx
    service_name: nginx
    message: "Hello from Ansible Variables"

  tasks:
    - name: Install package using variable
      apt:
        name: "{{ package_name }}"
        state: present
        update_cache: yes

    - name: Start service using variable
      service:
        name: "{{ service_name }}"
        state: started
        enabled: yes

    - name: Create custom web page
      copy:
        content: "{{ message }}"
        dest: /var/www/html/index.html
```

Save:

```
ESC → :wq
```

---

## **5. Run Playbook**

```
ansible-playbook -i inventory variable.yml --ask-become-pass
```

---

# 🔴 **Why this happens**

Common causes:

1. **Port conflict** → Another service (like Apache) already using port 80
2. **Nginx config error**
3. **Service not installed properly**
4. **WSL service issues (systemd behavior)**

👉 In your case, **most likely reason = Apache already running on port 80**

---

# ✅ **Step-by-Step Fix**

---

## **1. Check which service is using port 80**

Run:

```
sudo lsof -i :80
```

### If you see `apache2` → that’s the issue

---

## **2. Stop Apache (if installed earlier)**

```
sudo systemctl stop apache2
```

Also disable it:

```
sudo systemctl disable apache2
```

---

## **3. Start Nginx again**

```
sudo systemctl start nginx
```

Check status:

```
sudo systemctl status nginx
```

---

## **4. If still error → check logs**

```
sudo journalctl -xeu nginx.service
```

---

## **5. Alternative Fix (change port instead of stopping Apache)**

Edit Nginx config:

```
sudo vim /etc/nginx/sites-available/default
```

Change:

```
listen 80;
```

to:

```
listen 8081;
```

Then:

```
sudo systemctl restart nginx
```

Open:

```
http://localhost:8081
```
