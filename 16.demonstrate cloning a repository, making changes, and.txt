# 🧪 **Cloning a Repository, Updating Files, and Pushing Changes**

---

# 🎯 Aim

To clone a repository from GitHub, modify files locally, and push updates using Git.

---

# 📘 Theory

### 🔹 Clone

Creates a local copy of a remote repository.

### 🔹 Commit

Saves changes locally with a message.

### 🔹 Push

Uploads committed changes to GitHub.

---

### 🔹 Workflow

```text id="h9t4oj"
Clone → Modify → Add → Commit → Push
```

---

# ⚙️ Requirements

* Existing GitHub repository (name: `project-update-demo`)
* Git installed
* Internet connection

---

# 🧪 Procedure (Step-by-Step)

---

## 🔹 Step 1: Create Repository on GitHub

1. Open GitHub
2. Click **New Repository**
3. Enter:

   * Name: `project-update-demo`
4. Click **Create Repository**

---

### ✔️ Expected Output

* Repository page created with URL

---

## 🔹 Step 2: Clone Repository

```cmd id="uxq5t0"
git clone https://github.com/your-username/project-update-demo.git
cd project-update-demo
```

---

### ✔️ Expected Output

```text id="cykl2c"
Cloning into 'project-update-demo'...
```

---

## 🔹 Step 3: Create Initial File

```cmd id="brc4ho"
echo Initial content > file.txt
```

---

## 🔹 Step 4: Add and Commit

```cmd id="q5b29z"
git add file.txt
git commit -m "Initial commit: added file.txt"
```

---

## 🔹 Step 5: Push to GitHub

```cmd id="l7spbb"
git push origin main
```

---

### ✔️ Expected Output

* File appears in GitHub repository

---

## 🔹 Step 6: Modify File

```cmd id="pwovp9"
echo Added new update >> file.txt
```

---

## 🔹 Step 7: Check Status

```cmd id="u6c5uz"
git status
```

---

### ✔️ Expected Output

```text id="t1afgi"
modified: file.txt
```

---

## 🔹 Step 8: Add Changes

```cmd id="0wzgjg"
git add file.txt
```

---

## 🔹 Step 9: Commit Changes

```cmd id="1azr9r"
git commit -m "Updated file.txt with additional content"
```

---

## 🔹 Step 10: Push Updates

```cmd id="rztwgw"
git push origin main
```

---

### ✔️ Expected Output

```text id="l84b2s"
main -> main
```

---

## 🔹 Step 11: Verify on GitHub

Open repository → check:

* Updated `file.txt`
* New commit visible

---

# 📊 Result

* Repository created ✔️
* Repository cloned ✔️
* File created and modified ✔️
* Changes committed ✔️
* Updates pushed successfully ✔️

# 🧠 Key Concepts

| Command | Purpose           |
| ------- | ----------------- |
| clone   | Copy repo locally |
| add     | Stage changes     |
| commit  | Save changes      |
| push    | Upload to GitHub  |

# ❓ Viva Questions

**Q1. What is cloning?**
→ Copying repository to local system

**Q2. What is push?**
→ Uploading changes

**Q3. Why commit before push?**
→ To save changes locally first

---

# 🎯 Conclusion

This process demonstrates how to clone a repository, make updates, and synchronize changes with GitHub efficiently.
Short answer: **No, it’s not the same as 3.**
They look similar, but the **starting point and purpose are different**.

---

# 🔍 Key Difference

| Feature        | Repository Creation & Commit (3) | Clone, Modify & Push (16)     |
| -------------- | -------------------------------- | ----------------------------- |
| Starting point | No repo locally                  | Repo already exists on GitHub |
| First step     | `git init`                       | `git clone`                   |
| Remote setup   | Manual (`git remote add`)        | Automatic (comes with clone)  |
| Purpose        | Create new project               | Work on existing project      |

---

# 🧠 Conceptual Difference

### 🔹 Practical 3

```text
Local → Create repo → Connect → Push
```

👉 You are **starting from scratch**

---

### 🔹 Practical 16

```text
GitHub → Clone → Modify → Push
```

👉 You are **continuing an existing project**

---

# ⚠️ Important Understanding

* `git init` → used when project is **new**
* `git clone` → used when project **already exists online**

---

# 🎯 Real-Life Example

* Practical 3 → You create your own project
* Practical 16 → You join a team project and work on it

---

# ✅ Final Conclusion

👉 Both use:

* `add`
* `commit`
* `push`

BUT

👉 Difference lies in:

* **how the repository is created**
* **where the project comes from**
