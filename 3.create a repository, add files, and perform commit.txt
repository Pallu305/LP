# 🧪 **GitHub Repository Creation and Commit Operations**

---

# 🎯 Aim

To create a repository on GitHub, add files, and perform commit operations using Git with proper messages.

---

# 📘 Theory

### 🔹 Version Control System

Tracks changes in files and allows reverting to previous versions.

### 🔹 Git

A distributed version control system that works locally and remotely.

### 🔹 Repository

A storage location for project files and their history.

### 🔹 Workflow

```text
Working Directory → Staging Area → Repository
```

* **Working Directory**: where files are created/edited
* **Staging Area**: files prepared for commit
* **Repository**: saved snapshots (commits)

---

# ⚙️ Requirements

* Git installed
* GitHub account
* Command Prompt (CMD)

---

# 🧪 Procedure (Step-by-Step)

---

## 🔹 Step 1: Create Project Folder

```cmd
cd Desktop
mkdir github-demo
cd github-demo
```

### ✔️ Expected Output

```text
C:\Users\Pallavi\Desktop\github-demo>
```

---

## 🔹 Step 2: Initialize Git Repository

```cmd
git init
```

### ✔️ Expected Output

```text
Initialized empty Git repository in ...\github-demo\.git\
```

---

## 🔹 Step 3: Create a File

```cmd
echo Hello GitHub > file.txt
```

### ✔️ Expected Output

* `file.txt` is created

Check:

```cmd
dir
```

---

## 🔹 Step 4: Check Git Status

```cmd
git status
```

### ✔️ Expected Output

```text
On branch master

No commits yet

Untracked files:
  file.txt
```

---

## 🔹 Step 5: Add File to Staging Area

```cmd
git add file.txt
```

Check:

```cmd
git status
```

### ✔️ Expected Output

```text
Changes to be committed:
  new file: file.txt
```

---

## 🔹 Step 6: Commit the File

```cmd
git commit -m "Initial commit: added file.txt"
```

### ✔️ Expected Output

```text
[master (root-commit)] Initial commit: added file.txt
 1 file changed, 1 insertion(+)
 create mode 100644 file.txt
```

---

## 🔹 Step 7: Create Repository on GitHub

1. Open GitHub
2. Click **New Repository**
3. Enter:

   * Name: `github-demo`
4. Click **Create Repository**

### ✔️ Expected Output

* Repository page with URL displayed

---

## 🔹 Step 8: Connect Local Repository to GitHub

```cmd
git remote add origin https://github.com/your-username/github-demo.git
```

Verify:

```cmd
git remote -v
```

---

## 🔹 Step 9: Rename Branch to main

```cmd
git branch -M main
```

---

## 🔹 Step 10: Push to GitHub

```cmd
git push -u origin main
```

### ✔️ Expected Output

```text
Enumerating objects...
Writing objects...
To https://github.com/your-username/github-demo.git
 * [new branch] main -> main
```

---

## 🔹 Step 11: Verify Output on GitHub

* Open repository in browser
* You will see:

  * `file.txt`
  * Commit message

---

# 📊 Result

* Repository created successfully
* File added and committed
* Changes pushed to GitHub

# ⚠️ Common Errors

### 🔸 Authentication Failed

→ Use GitHub token instead of password

### 🔸 Branch Error

```cmd
git branch -M main
```

### 🔸 Remote Exists

```cmd
git remote remove origin
git remote add origin <url>
```

# 🧠 Key Concepts

* Git tracks file changes locally
* GitHub stores projects online
* `git add` → staging
* `git commit` → save snapshot
* `git push` → upload to remote
